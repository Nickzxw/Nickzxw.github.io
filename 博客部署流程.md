# Hexo 博客部署指南

## 环境说明

- 本地 `master` 分支存放的是**编译好的静态文件**（HTML/CSS/JS）
- 推送到 GitHub 后，GitHub Pages 会自动部署

---

## 每次发布新文章的流程

### 1. 本地编写文章

将 Markdown 源文件放在 `source/_posts/` 目录下（如果你有 Hexo 源码的话）。

**注意**：本仓库只有编译后的静态文件，没有 Hexo 源码。如果需要重新生成，请确保你有原始的 Hexo 项目。

### 2. 生成静态文件并替换

如果你有 Hexo 源码，执行：
```bash
hexo generate
```

生成的文件会出现在 `public/` 目录。把 `public/` 下的所有内容复制到当前仓库根目录，覆盖原有文件。

### 3. 提交并推送

```bash
# 添加所有更改
git add .

# 提交（写清楚改动内容）
git commit -m "你的提交信息"

# 推送到 GitHub
git push origin master
```

### 4. 等待自动部署

- GitHub 检测到 push 后会自动触发 Pages 构建
- 访问 **https://github.com/Nickzxw/Nickzxw.github.io/actions** 查看构建进度
- 通常 1-2 分钟内完成部署

## 常见问题

### 图片路径问题

文章中的图片应该使用**相对路径**，放在文章同目录下：

```html
<!-- 正确 -->
<img src="./image.jpg">

<!-- 错误（绝对路径会多一层目录） -->
<img src="文章名/image.jpg">
```

### 浏览器缓存

如果页面没更新，按 `Cmd + Shift + R` 强制刷新，或用无痕模式访问。

### GitHub Actions 没自动部署？

检查仓库 Settings → Pages → Source 确保是 `master` 分支。
