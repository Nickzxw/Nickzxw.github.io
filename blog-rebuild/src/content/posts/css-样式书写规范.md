# css-样式书写规范

## 样式规范

### 1、命名规范

- 类名使用小写字母
- 类名之间使用中划线 `-` 分隔
- 类名应该具有语义化

### 2、样式书写顺序

1. 定位（position、top、right、bottom、left、z-index）
2. 盒子模型（display、width、height、margin、padding、border）
3. 文本（font、color、text-align、line-height）
4. 背景（background、background-color）
5. 其他（border-radius、box-shadow、opacity）

### 3、样式缩写

- margin、padding 的值可以缩写
- border 的值可以缩写
- font 的值可以缩写

### 4、避免使用 filter

filter 属性会影响页面的渲染性能，尽量避免使用。

### 5、避免使用 expression

expression 属性会影响页面的性能，尽量避免使用。

### 6、避免使用 IE 的 hack

IE 的 hack 会影响代码的可维护性，尽量避免使用。

## 选择器规范

### 1、选择器命名

- 选择器命名应该具有语义化
- 选择器命名应该使用小写字母
- 选择器命名之间使用中划线 `-` 分隔

### 2、选择器嵌套

- 选择器嵌套的层级不应该超过 3 层
- 选择器嵌套应该保持简洁

### 3、选择器缩写

- 选择器的属性可以缩写

## 代码规范

### 1、缩进

- 使用 2 个空格缩进

### 2、注释

- 注释应该清晰明了
- 注释应该与代码保持一致

### 3、空行

- 代码块之间应该保持一行空行

### 4、换行

- 每行代码不应该超过 120 个字符
- 换行应该保持缩进

## CSS 预处理器

### 1、变量

- 使用变量存储常用的值
- 变量命名应该具有语义化

### 2、混合器

- 使用混合器封装重复的样式
- 混合器命名应该具有语义化

### 3、嵌套

- 嵌套层级不应该超过 3 层

## 最佳实践

### 1、避免使用 !important

!important 会影响样式的优先级，尽量避免使用。

### 2、避免使用通配符

通配符会影响页面的性能，尽量避免使用。

### 3、避免使用标签选择器

标签选择器的优先级较低，尽量使用类选择器。

### 4、使用 CSS 变量

CSS 变量可以方便地管理样式值，提高代码的可维护性。

### 5、使用 Flexbox 布局

Flexbox 布局可以方便地实现各种布局效果。

### 6、使用 Grid 布局

Grid 布局可以方便地实现复杂的布局效果。

## 响应式设计

### 1、媒体查询

- 媒体查询应该放在样式文件的底部
- 媒体查询应该按照屏幕尺寸的大小顺序排列

### 2、rem 单位

- 使用 rem 单位代替 px 单位
- rem 单位可以方便地实现响应式设计

### 3、弹性布局

- 使用弹性布局代替固定布局
- 弹性布局可以方便地适应不同的屏幕尺寸

## 性能优化

### 1、合并样式文件

- 将多个样式文件合并成一个
- 减少 HTTP 请求次数

### 2、压缩样式文件

- 压缩样式文件的大小
- 减少网络传输量

### 3、使用 CDN

- 使用 CDN 加速样式文件的加载

### 4、避免使用 @import

- @import 会阻塞页面的加载

### 5、使用硬件加速

- 使用 transform、opacity 等属性可以利用硬件加速
